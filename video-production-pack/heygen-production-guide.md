# HeyGen Production Guide

Use this workflow to create an AI-narrated product demo with a polished SaaS walkthrough style.

## Recommended Direction

For Growth Strategy Copilot, prioritize screen walkthrough over a large talking avatar. The product value is in the interface, metric logic, strategy scoring, and report output. If an avatar is used, keep it small or use it only for the intro and outro.

## 1. Prepare Visual Assets

Choose one of these approaches:

- Use a full screen recording of the product walkthrough.
- Use product screenshots for a presentation-style video.
- Combine short recordings with static screenshots for smoother pacing.

Recommended screens:

- Dashboard
- Diagnosis bottleneck
- Root Cause Hypotheses
- User Segmentation
- Strategy Priority Matrix
- Experiment card
- Executive Report
- English / Chinese switch

## 2. Add Script

1. Create a HeyGen video project.
2. Paste the English script from `full-demo-script-en.md`, or the Chinese script from `full-demo-script-zh.md`.
3. Split the script into scene-level blocks if needed.
4. Keep each scene focused on one product idea.

## 3. Choose AI Voice or Avatar

- Use a calm, professional AI voice.
- Avoid overly promotional or dramatic voice styles.
- If using an avatar, use it only in the opening or closing.
- Keep the main body focused on the product screen.

## 4. Add Captions

1. Use HeyGen's subtitle generation, or import SRT subtitles.
2. English: `subtitles-en.srt`.
3. Chinese: `subtitles-zh.srt`.
4. Keep subtitles short and readable.

## 5. Add On-Screen Captions

Use `on-screen-captions.md` for short overlay text. Recommended captions:

- Rules first, AI second
- Metric-driven bottleneck detection
- Transparent strategy prioritization
- From strategy to A/B test design
- Executive-ready reporting

## 6. Export

- Export as MP4.
- Use 1080p resolution.
- Keep the full demo under 3 minutes.
- Create a separate 45-60 second cut for social or portfolio previews.

## Quality Checklist

- Voiceover matches screen actions.
- The synthetic data limitation is mentioned.
- The demo does not imply live LLM integration.
- The avatar, if used, does not distract from the product.
- Subtitles are readable on desktop and mobile.
