# Deployment Notes

## 1. Current deployment mode

Growth Strategy Copilot is a Next.js App Router application designed for Vercel's default Next.js runtime. The current MVP uses a deterministic synthetic dataset, browser state and a mock structured analysis endpoint. It does not require a custom Vercel configuration, database or background service.

All application routes are defined under `app/`. The root route redirects to Dashboard, and direct access to analysis routes returns either the saved workspace or a guided empty state.

## 2. Why no API key is required

The current release does not call a real AI provider. Metrics, diagnosis, segmentation, strategy scoring, experiment plans and reports are generated by deterministic local logic and mock structured output.

No API key is required for local, preview or production deployments. The name `OPENAI_API_KEY` is reserved only as a possible future server-side environment variable; no value should be added to this repository.

## 3. What is stored in localStorage

The application stores two browser-local entries:

- `gsc-workspace`: Business Setup context, loaded user records, current analysis and data-source label.
- `gsc-language`: the selected English or Chinese presentation language.

The application handles missing or invalid storage by returning to safe defaults. If browser storage is blocked, the current in-memory session remains usable but will not persist after refresh.

## 4. Why data does not sync across devices

This demo uses browser localStorage, so project state is device-specific.

Opening the same URL on another device starts with a fresh local state. Vercel serves the application but does not store the visitor's workspace. No user authentication or cloud database is included in the current MVP.

Use **Start Demo with Fitness Sample** to reproduce the same deterministic demo flow on any supported device.

## 5. How to reset demo state

Select **Reset Demo** on Dashboard. This removes `gsc-workspace` from browser storage and restores the default Business Setup context. Language preference remains independent and can be changed through the sidebar language control.

If the interface cannot be opened, clear site storage for the deployment domain through the browser's privacy or developer settings.

## 6. Future AI API deployment

Future AI integration should preserve all metric, diagnosis, segmentation and scoring rules. The AI provider should receive structured computed facts and return schema-validated explanation or report wording.

Provider credentials must be added in **Vercel Project Settings → Environment Variables** and read only in server-side code. Do not expose a provider key through `NEXT_PUBLIC_*`, client bundles, repository files, documentation, screenshots or `vercel.json`.

Preview and production credentials should be scoped independently where appropriate. The application should retain deterministic fallback output when the provider is unavailable.

## 7. Security notes

- No credential is required or included in the current repository.
- `.env`, `.env.local`, `.env.*.local` and `.vercel` are ignored.
- Uploaded CSV data is processed in the browser and is not written to a cloud database by this MVP.
- Markdown exports are generated in a browser event and downloaded locally.
- localStorage is readable by scripts running on the same origin; do not use the MVP to process sensitive production data.
- Public deployment content should continue to use synthetic or appropriately anonymized demonstration data.

## 8. Known limitations

- Browser localStorage is not shared across devices, browsers or private sessions.
- Clearing site storage removes the saved workspace.
- There is no authentication, authorization, audit log or cloud project history.
- The 160-row sample supports workflow demonstration, not causal inference or production experiment sizing.
- A serverless instance does not preserve in-memory state between requests; the current product deliberately relies on browser state instead.
- Mobile charts are responsive but contain less room for labels than desktop views.
- The current industry interpretation remains optimized for fitness subscription analysis.
